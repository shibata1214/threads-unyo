# 編集学習：はるさんの手直しから文章ルールを更新する

はるさんが編集した投稿を受け取り、元の投稿と比較して「何をどう直したか」を分析し、
`05_writing.md` と `edit-history.md` に反映する。

---

## ステップ1：入力を受け取る

ユーザーから以下のどちらかの形式で受け取る：

**形式A：編集前後を両方もらう場合**
```
【元の投稿】
（Claudeが生成した投稿文）

【編集後】
（はるさんが直した投稿文）
```

**形式B：編集後のみもらう場合**
- 投稿IDが含まれていれば `post-queue.md` か `post-history.md` から元の投稿を探す
- IDがなければ「元の投稿はありますか？」と確認する
- 元の投稿が見つからない場合は、編集後の投稿だけから「文体・構成の特徴」を学習する

---

## ステップ2：差分を分析する

元の投稿と編集後を比較して、以下の観点で変更点を洗い出す：

### 分析観点
1. **1行目（フック）の変化**
   - 言葉を強くした？ 柔らかくした？
   - 数字・固有名詞を追加・変更した？
   - 文末の型（「〜だよ。」「〜して。」）を変えた？

2. **文体・リズムの変化**
   - 文を短くした？ 長くした？
   - 読点（、）の入れ方が変わった？
   - 改行の位置が変わった？

3. **言葉の選び方の変化**
   - AI感のある言葉を口語に直した？（例：「〜と言えるでしょう」→「〜だよ」）
   - 専門用語をわかりやすくした？
   - 感情表現を追加・変更した？

4. **構成の変化**
   - 情報の順序が変わった？
   - 何かを削除した？ 追加した？
   - CTAの書き方が変わった？

5. **全体の方向性**
   - より共感寄り？ 情報寄り？
   - より親しみやすく？ よりプロらしく？

---

## ステップ3：`edit-history.md` に記録する

以下のフォーマットで追記する：

```markdown
---
## 編集記録 YYYY-MM-DD（連番）

### 元の投稿
（元の投稿文をそのまま）

### 編集後
（はるさんの編集後の投稿文をそのまま）

### 変更の傾向（分析結果）
- フック：（何をどう変えた）
- 文体：（何をどう変えた）
- 言葉：（何をどう変えた）
- 構成：（何をどう変えた）
---
```

---

## ステップ4：`edit-history.md` の「蓄積された傾向まとめ」を更新する

冒頭の「蓄積された『はるさんの編集傾向』まとめ」セクションを最新に更新する。

過去の全編集記録をもとに、繰り返し変えられているパターンを箇条書きでまとめる。

例：
```
## 蓄積された「はるさんの編集傾向」まとめ

### フックについて
- フックの語尾は「〜だよ。」より「〜して。」「〜してほしい。」の命令型を好む
- 数字は「体感7度」より「7℃も」のように数字を先に持ってくるほうが好き

### 文体について
- 1文は短め（20〜30文字）を好む。Claudeが生成した長い文を分割する傾向あり
- 読点（、）を減らして体言止めにする傾向あり

### 言葉について
- 「〜することが大切」→「〜が大事」のように短くする
- カタカナ英語は減らす傾向（「スキンケア」→「保湿」など）
```

---

## ステップ5：`05_writing.md` を更新する

ステップ4で発見した傾向のうち、**3回以上繰り返されているもの** はルールとして `05_writing.md` に追記する。

追記場所：`05_writing.md` の末尾にある（またはなければ新設する）以下のセクション：

```markdown
---

## はるさんの編集傾向から学んだルール（自動更新）

※ `/learn-edit` を複数回実行した結果、繰り返し変更されるパターンをルール化したもの

- （ルール）
```

---

## ステップ6：結果をユーザーに報告する

以下の形式で報告する：

```
✅ 学習完了

【今回の主な変更点】
- （変更点1）
- （変更点2）
- ...

【蓄積傾向（N件の編集から）】
- （傾向1）
- （傾向2）
- ...

【今後の投稿に反映するルール】
（ルール化した内容。または「まだ3回に達していないため保留」）
```
