# アナリスト：数値分析→次の投稿改善

## ステップ1：データを読み込む
- `post-history.md` を読んで投稿記録とデータを確認する
- `analysis-latest.md` を読んで前回の分析結果を確認する

## ステップ2：数値を分析する

以下の観点で分析してください：

**パフォーマンス分析**
- いいね・コメント・保存数が多かった投稿TOP3
- 反応が少なかった投稿の共通点
- 最もクリックされた投稿（PR投稿の中で）

**ツリー誘導力分析**（ツリー投稿がある投稿のみ）
- 本投稿とツリー投稿それぞれのインプレッション数を比較する
- **ツリー到達率** = ツリーのインプレッション ÷ 本投稿のインプレッション × 100（%）を計算する
- 到達率の目安：
  - 30%以上 → 誘導文が強い・優秀
  - 15〜30% → 普通
  - 15%未満 → 誘導文が弱い・要改善
- 到達率が低い投稿は、本投稿内のどの言葉でツリーに誘導しようとしたか本文を読み返し、「なぜ読まれなかったか」を言語化する
  - 例：「続きはコメントに書いたよ」系の一言では弱い
  - 例：「コメントに○○の早見表を貼ったよ」のように**中身の具体的メリット**を示すと強い
- 到達率が高い投稿の誘導文を `06_references.md` の「ツリー誘導フレーズ実績」に記録する

**パターン分析**
- 伸びた投稿に共通するフック型は何か
- 伸びた投稿のテーマジャンルは何か
- 投稿時間帯による反応の差はあるか

**ワード分析**
- 伸びた投稿に含まれていたキーワード（例：「癇癪」「寝かしつけ」など感情・悩みワード）
- 商品名・固有名詞が入った投稿の反応は上がっているか
- 反応が少ない投稿に多いワードやフレーズ
- 「保存されやすいワード」「コメントを引くワード」を区別して記録する

## ステップ3：改善ポイントを抽出する

- 今後増やすべき投稿の型・テーマ
- 今後減らすべき投稿の型・テーマ
- 次週の投稿で試すべき新しいアプローチ
- 反応が良かったワード・悪かったワードを `06_references.md` の「🔤 反応を引いたワード実績」テーブルに追記する
- ツリー到達率が15%未満の投稿は、改善すべき誘導文の具体的な言い換え案を1〜2本提案する

## ステップ4：next-topics.md を更新する

分析結果をもとに `next-topics.md` の「今週のテーマ候補」を更新してください。
伸びたテーマの横展開や、コメントで質問が多かったネタを優先する。
