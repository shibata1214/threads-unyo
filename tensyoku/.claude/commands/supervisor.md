# スーパーバイザー：運用全体を監視・診断する

エージェントが正常に動いているかを確認して、
問題があれば原因と対策を報告します。

## ステップ1：全ファイルを読み込む

以下のファイルを全て読んでください：
- `post-history.md`（投稿履歴・直近の投稿日・データ取得状況）
- `post-queue.md`（投稿待機列・残件数）
- `next-topics.md`（ネタストック・テーマ数）
- `analysis-latest.md`（最新分析結果・更新日）
- `06_references.md`（リサーチストック・更新日）
- `07_ng-rules.md`（ルール確認）
- `01_profile.md`〜`07_ng-rules.md`（ナレッジファイルの中身を確認）

## ステップ2：5項目チェックリストを実行する

### チェック①：投稿が止まっていないか
- `post-history.md` の最新投稿日を確認
- 2日以上投稿がなければ → **⚠️ 投稿停止中**
- 今日の投稿数が3件を超えていれば → **⚠️ 投稿上限超過（凍結リスク）**

### チェック②：ネタストックは足りているか
- `next-topics.md` のテーマ数を確認
- 3つ以下なら → **⚠️ ネタ枯渇気味**
- 0個なら → **🔴 ネタ枯渇（ライターが動けない）**

### チェック③：投稿キューは適切か
- `post-queue.md` の未投稿数を確認
- 14件以上溜まっていたら → **⚠️ キュー溜まりすぎ**（2週間分以上）
- 0件なら → **⚠️ キュー空（今日投稿できない）**

### チェック④：メトリクス取得はできているか
- `post-history.md` で `metrics_fetched: false` の件数を確認
- 5件以上未取得なら → **⚠️ メトリクス未取得が多い**
- Threads投稿IDが空の投稿があれば → **⚠️ 投稿IDが記録されていない**

### チェック⑤：ナレッジファイルは充実しているか
- `01_profile.md`〜`07_ng-rules.md` の各ファイルの中身を確認
- 空・未記入のセクションがあれば → **⚠️ ナレッジ不備**
- `06_references.md` のリサーチ追加が1週間以上ないなら → **⚠️ リサーチ停滞**

## ステップ3：週次チェックリスト

- [ ] 今週の投稿、誰の何を解決したか説明できるか
- [ ] 投稿が伸びなかった場合、原因を3つ挙げられるか
- [ ] 1週間後のテーマが最低5個あるか
- [ ] 反応が良かった投稿を横展開しているか
- [ ] 有益投稿：PR投稿＝7：3の比率を守れているか
- [ ] 1日3件以上の投稿をしていないか
- [ ] PR投稿に直接URLを貼っていないか
- [ ] NGルール違反はないか

## ステップ5：Notionに保存する

環境変数 NOTION_API_KEY と NOTION_TENSYOKU_REPORT_PAGE_ID を使って、
指定ページに運用レポートを子ページとして保存してください。

```bash
TODAY=$(date '+%Y-%m-%d')
curl -s -X POST https://api.notion.com/v1/pages \
  -H "Authorization: Bearer ${NOTION_API_KEY}" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d "{
    \"parent\": {\"page_id\": \"${NOTION_TENSYOKU_REPORT_PAGE_ID}\"},
    \"properties\": {\"title\": [{\"text\": {\"content\": \"運用レポート $TODAY\"}}]},
    \"children\": []
  }"
```

## ステップ7：日本語でわかりやすく報告する

チェック結果と改善提案を、みいさにわかりやすく日本語で報告してください。

**ステータス判定基準：**
