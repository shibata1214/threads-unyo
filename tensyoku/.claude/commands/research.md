# リサーチャー：ネタ収集・競合分析

## 使い方
URLを渡してください。X・Threads・インスタ・YouTubeのURLに対応しています。
URLなしで実行した場合は、看護師転職系キーワードで自動検索します。

## ステップ1：URLの内容を取得する

渡されたURLにアクセスして以下を取得してください：
- 投稿・動画のタイトルや本文
- 反応数（いいね・コメント・保存・再生数）
- 主なコメントの内容（質問・共感が多いもの）

URLがない場合は、以下のキーワードで検索して上位コンテンツを分析する：
- 「看護師 転職 ママ」
- 「看護師 夜勤なし キャリア」
- 「クリニック 転職 看護師」
- 「転職エージェント 看護師 比較」

## ステップ2：内容を分析する

以下の観点で分析してください：
- どんなフック型を使っているか
- テーマ・ジャンルは何か
- なぜ伸びているか（仮説）
- みいさのアカウントに転用できる切り口はあるか
  （病棟未経験・夜勤ゼロ・看護師ママの視点で使えるか）

## ステップ3：06_references.md に追記する

`/Users/mina/看護師転職/06_references.md` の「投稿ネタストック」に以下フォーマットで追記してください：

```
### （追加日）リサーチ追加
- 元ネタ：（URLまたはアカウント名）
- テーマ：（内容）
- 転用アイデア：（みいさの病棟未経験・看護師ママ視点でどう使えるか）
- 使えるフック：（具体的な1行目の案）
- 反応数：（いいね・コメント等）
```

## ステップ4：next-topics.md にネタを追加する

転用できそうなネタを `/Users/mina/看護師転職/next-topics.md` に追記してください：

```
### （日付）リサーチから追加
- テーマ：（投稿化できるテーマ）
- 切り口：（みいさの視点でどう使うか）
- フック案：（1行目の案）
- 参考元：（URL or アカウント名）
```

## ステップ5：GitHubにプッシュ

```bash
cd /Users/mina/看護師転職
git add 06_references.md next-topics.md
git commit -m "リサーチ追加：$(date '+%Y-%m-%d')"
git push origin main
```

## ステップ6：結果を報告する

- 分析したコンテンツ数
- 特に使えそうなネタTOP3
- みいさにしか語れない差別化ポイントがあれば強調する
