# アナリスト：数値分析→次の投稿改善

## ステップ1：データを読み込む
- `post-history.md` を読んで投稿記録とデータを確認する
- `analysis-latest.md` を読んで前回の分析結果を確認する
- `06_references.md` を読んで競合・バズパターンを確認する

## ステップ2：数値を分析する

以下の観点で分析してください：

**パフォーマンス分析**
- いいね・コメント・保存数が多かった投稿TOP3
- 反応が少なかった投稿の共通点
- PR投稿の中でエンゲージメントが高かったもの（エージェント紹介の効果測定）

**パターン分析**
- 伸びた投稿に共通するフック型は何か
- 伸びた投稿のテーマジャンルは何か（体験談型・比較型・ノウハウ型 など）
- 投稿時間帯による反応の差はあるか
- 有益投稿とPR投稿でエンゲージメント率に差はあるか

## ステップ3：改善ポイントを抽出する

- 今後増やすべき投稿の型・テーマ
- 今後減らすべき投稿の型・テーマ
- 次週の投稿で試すべき新しいアプローチ
- 反応が良かったPR投稿のエージェントは「7回訴求リスト」に追加する

## ステップ4：next-topics.md を更新する

分析結果をもとに `next-topics.md` の「今週のテーマ候補」を更新してください。

**必ず実行すること：**
- 反応が良かった投稿（いいね・保存・コメントが多い）→「リメイク候補」として追加
  （型を変える・ターゲットを変える・数字や事例を別のものにする）
- 反応が良かったPR投稿 → 「7回訴求リスト」として追加
  （同じエージェントを切り口を変えて7パターン訴求する。残り本数も記載）
- コメントで質問が多かったネタ → 優先テーマとして追加

## ステップ8：Notionに保存する

環境変数 NOTION_API_KEY と NOTION_TENSYOKU_ANALYSIS_PAGE_ID を使って、
指定ページに分析レポートを子ページとして保存してください。

```bash
TODAY=$(date '+%Y-%m-%d')
curl -s -X POST https://api.notion.com/v1/pages \
  -H "Authorization: Bearer ${NOTION_API_KEY}" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d "{
    \"parent\": {\"page_id\": \"${NOTION_TENSYOKU_ANALYSIS_PAGE_ID}\"},
    \"properties\": {\"title\": [{\"text\": {\"content\": \"分析レポート $TODAY\"}}]},
    \"children\": []
  }"
```
