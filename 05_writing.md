# 文章ルール・トンマナ

## 基本口調
- タメ語（〜だよ、〜だよね、〜してみて）
- 親しみやすく、でも信頼できる先輩ママ＆専門家として
- 難しい医療用語は使わない。使う場合はすぐ説明を添える
- ママ友に話しかけるような感覚で書く

## ★ペルソナ優先ルール（2026-04-18追加）

**投稿の主役は「はる」ではなく「読んでいるママ」。**

### 自分語りNG
- 「今日〇〇した」「最近〇〇してた」など日常エピソードを主役にしない
- はるの感情・出来事で始まる投稿は作らない
- 体験談を使うときも「あなたに役立つ情報」として届けること

### ペルソナが求めていること（02_target.mdより）
- 子どもの体の異変への不安を解消したい
- 肌荒れ・発熱など「何を使えばいいか」を知りたい
- 育児のメンタル限界に共感・出口を見せてほしい
- 信頼できる専門家から「わかりやすく」教えてほしい

### 書くときの問い
投稿を作る前に「**このペルソナの何の悩みを解決する投稿か？**」を必ず確認する。
悩みに直接刺さっていなければ書き直す。

## 投稿の構成
1行目：フック（下記の型から必ず選ぶ）
2〜4行目：医療知識 or リアル体験をもとに解説
5〜6行目：具体的なアドバイス or 商品紹介
最終行：保存・フォローを促すCTA

## 文字数
200〜350文字

## 1行目フックの徹底ルール

**フックは投稿の命。以下を必ず守る：**
- 具体的な数字か固有名詞を入れる（「体感7度」「38.5℃」「週1回」など）
- 意外性・逆説・禁止のどれかを必ず含める
- 「〜なの、知ってた？」「怖いことを言うけど」「〜じゃない、〜なんだよ」系が強い
- 絶対に避ける：「こんにちは」「はるです」「今日は〇〇について」の自己紹介出し

**引きが強い言葉リスト（積極的に使う）：**
| カテゴリ | 使える言葉 |
|---|---|
| 禁止・逆説 | 「じゃない」「しないで」「違う」「間違い」「実は逆」「逆効果」 |
| 緊迫感 | 「怖いことを言うけど」「命に関わる」「もう遅い」「危ない」「知らんと一家全滅」「正直ゾッとした」「ビビり散らかした」 |
| 数字 | 「体感7度」「週1回」「38.5℃」「10℃以上」「30%」「〇人以上見てきた」「〇歳以下」 |

## 投稿の型（10パターン）※必ずどれかを選んで使う

### 【型①】宣言型
フック：「〇〇に絶対に知ってほしいことがある。」
構造：宣言 → 内容・理由 → 具体例 → CTA
例フック：「保育園に入ったばかりのママに絶対に知ってほしいことがある。」

### 【型②】否定→反転型
フック：「〇〇じゃない、〇〇なんだよ。」
構造：常識の否定 → 正しい情報 → なぜそうなのか → 実践法
例フック：「子どもの夜泣きは『疲れすぎ』じゃなくて、脳の成長サインなんだよ。」

### 【型③】警告型（最も拡散されやすい）
フック：「怖いことを言うけど、」「〇〇の人は今すぐ確認して。」
構造：危機感の提示 → 根拠（数字・統計） → 対処法 → 保存CTA
例フック：「怖いことを言うけど、慣らし保育中がSIDSで一番多い。」

### 【型④】専門家視点型
フック：「看護師として正直に言うと、」「看護師目線で見てると、」
構造：立場の明示 → 本音・内側の話 → 読者への応用 → フォローCTA
例フック：「看護師として正直に言うと、子どもの発熱で一番やってはいけないことがある。」

### 【型⑤】数字インパクト型
フック：「赤ちゃんは大人より体感**7度**暑い。」「〇〇が**38.5℃**を超えたとき、」
構造：意外な数字 → なぜその数字か → 親がすべき対応 → CTA
例フック：「ベビーカーの地面から50cmの温度、大人より10℃以上高いって知ってた？」

### 【型⑥】体験談型
フック：「2人育てて気づいたこと。」「〇年看護師やって初めてわかった。」
構造：体験の提示 → 当時の状況 → 気づき → 読者へのメッセージ
例フック：「2人育てて気づいたことは、4〜5月の子どもの荒れは『わがまま』じゃないってこと。」

### 【型⑦】問いかけ型
フック：「〇〇って知ってた？」「これ、知ってる人いる？」
構造：問い → 「実は…」の答え → 解説 → 保存・フォローCTA
例フック：「子どもが『喉渇いた』と言ったとき、もう脱水が始まってるって知ってた？」

### 【型⑧】命令・指示型
フック：「間髪入れずに〇〇してください。」「今すぐやること↓」
構造：強めの指示 → 理由 → 箇条書きアクション → やさしいCTA
例フック：「間髪入れずに『もちろんいいよ』と受け止めてください。」

### 【型⑨】共感型
フック：「〇〇のとき、こうなるよね。」「これわかる人いる？」
構造：共感フック → あるあるエピソード → 実は〇〇だった → 温かいCTA
例フック：「帰宅後に泣く、癇癪、夜泣き復活——これ全部、子どもからのSOSだよ。」

### 【型⑩】日常体験→学び型
フック：「〇〇に行ったら〇〇があって、」（日常の一場面 → 専門知識へ橋渡し）
構造：日常の一場面 → 「実はこれ〇〇なんだよね」 → 専門解説 → CTA
例フック：「今日GWの公園連れて行ったんだけど、地面が思ったより全然熱くて。」

## 意外性を出すための「内容の逆張りアイデア」

- 「常識と思われていることを覆す」：「〇〇は良い、は実は間違い」
- 「ポジティブなものの裏側を見せる」：「順調に見えてた〇〇が実はリスク」
- 「当たり前の行動に警鐘を鳴らす」：「みんなやってるけど〇〇はNG」
- 「専門家しか知らない事実を出す」：「看護師は内心〇〇と思ってる」
- 「タイミングのずれを指摘する」：「〇〇したときはもう遅い」

## ツリー投稿の設計ルール（実績から導出・2026-04-18）

**ツリーはリポスト（拡散）を生む。必ず設計して使う。**

実績：#8000/#7119投稿で本投稿エンゲ率0.25% → ツリーエンゲ率1.67%（6.7倍）
ツリーのリポストは本投稿の4倍。「早見表」として拡散された。

### 情報系投稿のツリー構成（必須）
- 本投稿：フック＋「怖いシーン」or「あるある」で感情を揺らす（200〜300文字）
- ツリー：箇条書き早見表 or 使い分けチャート。「✅/❌」「→」「【】」で視覚的に整理
- ツリー末尾のCTA：「今すぐスマホに登録してね」「保存しておいてね」

### ツリーに向いているコンテンツ
- 「使い分け・判断基準」系（#8000/#7119、発熱の受診サイン など）
- 「やっていい・NGリスト」系（ハチ刺され対処、キズパワーパッドなど）
- 「手順・フロー」系（緊急時の対応順序など）

### ツリーに向いていないコンテンツ
- 体験談・感情系（本投稿で完結させる）
- PR投稿（ツリーに楽天リンクを貼る用途では使うが、まとめは不要）

## 楽天PR投稿ルール

- 「楽天ROOMにリンク」は使わない（楽天ROOM未作成）
- コメント欄にURLを貼る形式（ツリー投稿）
- 「リンクはコメント欄に貼ってるよ🛍️」とメイン投稿に書く
- 必ず「#PR」を入れる
- 1商品につき約7回（2週間かけて）訴求する
  - 最初の投稿：商品紹介（型③警告型 or 型⑤数字型）
  - 2〜3本目：体験談（型⑥体験談型）
  - 4〜5本目：別の切り口（型②否定反転型 or 型①宣言型）
  - 6〜7本目：まとめ・違うターゲット層へ

### ★「同一商品×切り口分解」戦略（7回訴求の核心）

**売れる商品を見つけたら、「誰のどんな悩みに効くか」を複数考えて投稿を量産する。**

同じ商品でも切り口が違えば別の投稿になり、異なる層に刺さる。
ライバルと同じ商品でも切り口が違えば「パクリ」にならない。

#### 切り口の3軸

| 軸 | 内容 | 例（子ども用踏み台） |
|---|---|---|
| **ターゲット** | 刺さる悩みが違うママを変える | 自立させたいママ／体がしんどいママ／朝忙しいママ |
| **型** | 同じ情報でも伝え方を変える | 警告型→体験談型→数字インパクト型 |
| **シーン** | 使う場面・タイミングを変える | 朝の支度／お風呂前／外出前 |

#### 切り口の考え方（1商品投稿前に必ずやる）
1. この商品は「誰の」「どんな悩み」を解決するか？ → 3パターン以上書き出す
2. それぞれに対応する投稿の型を1つ選ぶ
3. フックに「ターゲット限定ワード」を入れる（例：「腰がしんどいママへ」「朝5分でも短縮したいなら」）

#### 切り口プランの例（子ども用踏み台）
- ❶「これで子どもが自分で手洗いできるようになった！」→ 自立を促したいママ
- ❷「毎回抱っこで腰やばかったけど、これで解放された」→ 体力的にしんどいママ
- ❸「朝の支度で『ママやって〜』が減って10分短縮できた」→ 朝忙しいママ
- ❹「買う前に後悔してほしくない、踏み台選びで失敗した話」→ 失敗を避けたいママ（警告型）
- ❺「保育園に踏み台を送ったら先生に感謝された話」→ 保育園でも使えることを知らないママ
- ❻数字インパクト型：「3歳が自分で手洗いできるまで1日でいけた理由」→ 効果を具体的に知りたいママ
- ❼まとめ：「踏み台を買って変わったこと7つ」→ 未検討層への総まとめ

#### ライターへの指示
商品紹介投稿を作るとき、**最初の1本を書く前に「切り口プラン」を7パターン書いてから**投稿文を作り始める。
`next-topics.md` の「リメイク候補」セクションに、商品名と切り口リストをセットで記録しておく。

## はるさんの編集傾向から学んだルール（自動更新）

※ `/learn-edit` を複数回実行した結果、繰り返し変更されるパターンをルール化したもの（5件）

### ツリー分割（コメント欄への引き込み）
- **文が途中で終わる形で引き込む**：「一番怖いのが、」「それを解決できる方法は、」「でも看護師として正直に言うと、」
- リスト直前や段落の区切りで分割するのはNG。「気になって続きを見たくなる」一文の途中で切ること

### 感情の強さは文字で表す
- 「めっっっちゃ」「ほんっっっとに」のように促音・長音を3〜4つ重ねる
- 「！！」を恐れずに使う。「〜よ。」より「〜！」や「〜！！」を優先

### 口語化のルール
- 「2人育てて気づいたことは、〜ってこと。」→**「2人育てて気づいたんだけど、〜。」**
- 「〜のが最も危ない特徴」→「〜のの」「〜なんだよ」
- 「〜ことが大切」→「〜が大事」「〜が大切だよ」

### CTAの使い方
- 「保存しておいてね」を毎回入れない。連続使用はNG
- 自然な締め（「GWはハードル下げてこ！！」「家族全員で確認しておいてね🌿」）に変える
- 感情系・体験談型の投稿はCTAなしで終わってもよい

### 絵文字の使い方
- 🌿はデフォルトにしない。感情に合った絵文字を使う
- ほっこり・感動 → 🥹
- 笑い・共感 → ww、w
- 情報系の締め → 🌿でOK

### 自分の体験・本音を末尾に添える
- コメント欄の末尾に「ちなみに我が家は〜」「私はこういうの使ってた」など自分の話を短く追加
- 読者との距離を縮める「自分ごと」で終わると親しみやすくなる

### PRリンクの貼り方
- 「リンクはコメント欄に貼ってるよ🛍️」は不要
- **「pr https://a.r10.to/xxxxx」形式で直接貼る**
- 複数商品は「お出かけが多い人はこれ↓」「そこまでいらない人はこっち↓」で分けて並べてOK

### リスト形式
- 5項目以上 → **番号付きリスト（①②③）**を使う
- 行動リスト → ✔️のまま
- 横並びテキストで情報を列挙するのはNG

### フックはリストから選ぶ＋結論を一文で続ける
- Claudeが作った否定形・命令形フックより、05_writing.mdのバズフックリストにある既存フックを使う
- 「マジで広がってほしいこと。」は特に強い。フック後すぐに「一番伝えたいことを短く1文」で続けること
- 形式：「（フック）。（結論1文）。（ターゲット・状況）。」

### ターゲットはシーンで絞る
- 「子どもを持つ親」より「GWに子どもを預ける人」「保育園の朝に迷うママ」のように状況を絞ると刺さる
- フックの直後にターゲットシーンを明示する

### 単体投稿をツリー構成に組み替える判断
- 情報量が多い場合・チェックリストがある場合は単体投稿でなくツリー構成にする
- 「これ、危ない理由があって、」「一番怖いのが、」など文の途中で切ってコメント欄へ誘導
- 本投稿は「感情・エピソード・フック」、コメント欄は「情報・リスト・CTA」で役割分担

### 医学表現は断定より「第一選択」「との関連が指摘されている」
- 「〇〇のみ」「〇〇だけ」→「第一選択は〇〇」に修正
- 「〜病につながる」→「〜との関連が指摘されている」に修正
- 商品名（カロナール・こどもバファリンなど）は原則削除。成分名に留める

### CTAは行動を一つに絞って強く締める
- 複数CTAを並べない（「確認してみて」「保存しておいてね」を同時に入れない）
- 「預ける前に一言伝えて！！」のように具体的な行動一つを！！で締める
