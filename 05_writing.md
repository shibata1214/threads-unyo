# 文章ルール・トンマナ

## 基本口調
- タメ語（〜だよ、〜だよね、〜してみて）
- 親しみやすい先輩看護師の感覚
- 難しい転職用語は使わない。使う場合はすぐ説明を添える
- 同期・同職の仲間に話しかけるような感覚で書く

---

## 【最重要】発信の信頼性ルール

**自分が実際にやったこと・経験したことしか断言しない。**

- 「面接9割合格した私がやったこと」→ OK（実体験）
- 「これをすれば絶対内定取れる」→ NG（断言しすぎ）
- 「病棟は〇〇らしい」（病棟経験がないのに）→ NG（未経験ジャンルを断言）
- 「転職サイトの〇〇は〇〇すれば使える」→ OK（実際に使った経験から）

病棟経験がないことは隠さない。「クリニック専門だからこそ知っていること」として積極的に活かす。

---

## 【最重要】ペルソナ優先ルール

**投稿の主役は「はる」ではなく「読んでいる看護師」。**

### ペルソナが求めていること（02_target.mdより）
- 転職で失敗したくない
- クリニックのリアルを知りたい
- 面接・書類・条件交渉の具体的な方法を知りたい
- 育児と両立できる職場の選び方を知りたい
- 自分と似た境遇の先輩の話を聞きたい

### 書くときの問い
投稿を作る前に「**この投稿はターゲット看護師のどの悩みを解決するか？**」を確認する。

---

## 投稿の構成
1行目：フック（下記の型から必ず選ぶ）
2〜4行目：体験・事実・解説（実体験ベースで）
5〜6行目：具体的なアドバイス or 転職サイト紹介
最終行：保存・フォロー・転職サイト登録を促すCTA

## 文字数
200〜350文字

## ★【投稿前チェックリスト】書いたら必ず確認する

### 冒頭1行目チェック（3つ全部クリアしてから次へ）

| チェック | NG例 | OK例 |
|---|---|---|
| 固有名詞が入っているか | 「大手企業」「転職サイト」「ある職場」 | 「レバウェル看護」「整形外科クリニック」「産婦人科」 |
| 数字が入っているか | 「すごく伸びた」「かなり高い」「たくさん」 | 「3ヶ月で1万人」「時給200円アップ」「面接9割合格」 |
| ターゲットが明確か | 「ママ」「看護師さん」「転職したい人」 | 「2児ワンオペ看護師」「30代クリニック転職希望」「ブランク1年の看護師」 |

→ 1つでも欠けていたら書き直す。

### 本文チェック（3つ全部クリアしてから完成）

| チェック | NG例 | OK例 |
|---|---|---|
| 権威性が入っているか | 「転職は大変です」（根拠なし） | 「6ヶ月続けてわかった」「3職場経験して気づいた」「実際に試した結果」 |
| 体験談・具体的な数字があるか | 「条件を確認することが大事」（抽象的） | 「私がやったのは面接でこの一言だけ。時給が200円上がった」 |
| 末尾が行動を促しているか | 「参考になれば嬉しいです」（ぼんやり） | 断定（「これだけやれば変わる」）か問いかけ（「知りたい人いる？」）で終わる |

→ 抽象的な説明だけで終わっている段落があれば具体的な数字か体験談を足す。

## 改行ルール（スマホ表示対策）
- 文中に不要な改行を入れない
- 67文字前後を目安に、文末（「。」直後）で1回改行
- 目安：1段落＝60〜70文字、最大2〜3段落で本投稿を構成する
- 箇条書きは1項目ごとに改行

## 1行目フックの絶対ルール

**フックは投稿の命。以下を必ず守る：**
- 具体的な数字か固有名詞を入れる（「9割合格」「時給200円」「3職場」など）
- 意外性・逆説・暴露のどれかを必ず含める
- 「えっ、知らなかった！」か「わかる！！」のどちらかを引き出す
- 絶対に避ける：「こんにちは」「はるです」「今日は〇〇について」の自己紹介始まり

### ★★★【最重要】フックの強さチェック★★★

1行目を書いたら必ず問う：**「パッと見てこの投稿を開く理由があるか？」**

読者の頭に瞬時に浮かぶべき感覚：
- 「**自分は見た方がいい投稿だ**」
- 「**有益そうだ**」
- 「**この投稿見たら損しなさそう**」

これが浮かばないフックは書き直す。

#### 強いフックの5パターン（必ずどれかで書く）

| パターン | 構文 | 例 |
|---|---|---|
| **①直接行動指示** | 「〇〇で悩んでる人、これやって。」 | 「クリニック転職で面接落ちてる人、これやって。」 |
| **②私が教える宣言** | 「〇〇した私が〇〇を教える。」 | 「3職場経験した私が面接の通し方を教える。」 |
| **③損しない保証** | 「〇〇する前にこれだけ読んで。」「知らないと損する〇〇。」 | 「転職サイト登録する前にこれだけ読んで。」 |
| **④衝撃数字先出し** | 「〇〇が〇〇だった話。」（数字を1行目に） | 「時給200円上がった。やったのは一言だけ。」 |
| **⑤暴露・ぶっちゃけ** | 「〇〇、正直に言う。」「〇〇の本音。」 | 「クリニック転職、失敗する人の共通点を正直に言う。」 |

#### ❌ 弱いフックの例（書かない）

- 「クリニック転職についてお話しします」→ 開く理由がゼロ
- 「転職を考えている方へ」→ ぼんやりしすぎ
- 「看護師さん向けの情報です」→ 何が得られるかわからない
- 「最近気づいたことがあって」→ 前置きすぎて引きがない

**引きが強い言葉リスト（積極的に使う）：**
| カテゴリ | 使える言葉 |
|---|---|
| 暴露・裏側 | 「実は」「裏事情」「本音で言うと」「教えてくれなかった話」「知らなかった」「ぶっちゃけると」 |
| 警告・禁止 | 「やめて」「NG」「危ない」「後悔する」「損してる」「だまされてた」 |
| 数字 | 「9割合格」「時給200円アップ」「3職場経験」「11年目」「1年目で」 |
| ビフォーアフター | 「〇〇だった私が」「〇〇して変わったこと」「転職前と後で全然違う話」 |
| 共感 | 「わかる人いる？」「これあるあるだと思うけど」「同じ経験したことある？」 |
| 専門家本音 | 「看護師として正直に言うと」「11年やってきてわかること」「クリニック3職場経験して言えること」 |
| ターゲット限定 | 「クリニック転職考えてる人へ」「ママ看護師さんに伝えたい」「病棟に疲れた人に読んでほしい」 |

**★フックまとめ（実績・競合から収集）：**
- 「看護師として正直に言うと、」
- 「転職サイトに登録する前に知っておいてほしいことがある。」
- 「病棟からクリニックに転職して〇年。リアルを言う。」
- 「面接9割合格してきた私がやってたこと。」
- 「時給交渉で200円アップした話。」
- 「クリニック転職したいなら、これだけは絶対に確認して。」
- 「転職エージェントが教えてくれなかったこと。」
- 「3職場経験した看護師として正直に言う。」
- 「ブランクあっても採用された理由。」
- 「これ知らずに転職すると後悔する。」
- 「看護師11年目が選んだ転職サイトはこれ。」
- 「クリニック看護師のリアルな給料、言ってもいい？」
- 「面接で落ちる人の共通点、全部同じだった。」
- 「転職サイト〇社使って気づいたこと。」
- 「育休なしで辞めた私が次の職場で評価された理由。」
- 「夜勤なしでも年収〇〇万はいける。」
- 「クリニック面接で聞かれること、ほぼ決まってる。」
- 「転職を迷ってるなら、とりあえずこれだけやって。」

## 投稿の型（8パターン）※必ずどれかを選んで使う

### 【型①】体験談・ビフォーアフター型
フック：「〇〇だった私が〇〇になった話。」
構造：当時の状況 → 転機 → 変化 → 読者へのメッセージ
例：「新卒でクリニックを選んだ私が、11年後に院長から言われた言葉。」

### 【型②】暴露・裏側型
フック：「〇〇が教えてくれなかったこと。」「〇〇の裏事情。」
構造：一般に知られていない情報 → 理由・メカニズム → 読者が取るべき行動
例：「転職サイトの担当者が言わない、クリニック求人の見極め方。」

### 【型③】警告型
フック：「これ知らずに転職すると後悔する。」「〇〇は絶対やめて。」
構造：危機感の提示 → 具体的なNG例 → 正しい方法 → 保存CTA
例：「面接でこれを言うと確実に落ちる。クリニック面接11年目の話。」

### 【型④】実践攻略型
フック：「〇〇するなら、これだけやれば大丈夫。」
構造：課題の提示 → 具体的なステップ → 注意点 → CTA
例：「クリニック転職の面接対策、これだけやれば9割受かる。」

### 【型⑤】数字インパクト型
フック：「〇〇年で〇〇円上げた話。」「〇割が知らない〇〇。」
構造：数字で驚かせる → なぜその数字か → 読者への応用 → CTA
例：「時給交渉で200円アップした話。やったのはたった2つ。」

### 【型⑥】比較・まとめ型
フック：「〇〇と〇〇、正直どっちがいい？」
構造：AとBの比較提示 → それぞれのメリット・デメリット → 読者への提案
例：「レバウェル、ナースではたらこ、看護roo、3社使って感じた違い。」

### 【型⑦】共感型
フック：「これわかる人いる？」「〇〇のとき、こうなるよね。」
構造：あるある状況の提示 → 「実はこれって〇〇なんだよね」 → 解決策
例：「転職活動してると担当者から毎日LINEくる件、わかる人いる？」

### 【型⑧】問いかけ型
フック：「〇〇って考えたことある？」「〇〇、どうしてる？」
構造：問いかけ → 「実は〇〇な人が多い」 → 解説 → 行動CTA
例：「転職サイトに登録したあとの断り方、考えたことある？」

### 【型⑨】権威・信頼構築型（実績→フォロー誘導）
フック：「〇〇した私が〇〇してきた話。」「〇〇な実績がある私が発信しています。」
構造：実績・経歴の提示 → それが読者にとってなぜ価値あるか → フォロー・保存CTA
例：「新卒からクリニック専門・面接9割合格・ママ看護師11年。このアカウントで話すこと。」
**使いどき：** アカウント立ち上げ期・久々の投稿前・フォロワーが増えたタイミング

### 【型⑩】需要確認型（いいね確認投稿）
フック：「〇〇の方法知りたい人いる？需要あれば投稿するね。」
構造：テーマをチラ見せ → 「いいね・コメントで教えて」 → 期待感を作る
例：「子どもがいながら面接100%合格してきた秘訣、需要あれば投稿する。いいねで教えて。」
**使いどき：** ネタの需要を測りたいとき・エンゲージメントを上げたいとき・投稿への期待感を高めたいとき
**注意：** 必ず反応があったら本編を投稿する。需要確認だけで終わらせない。

## ★エピソードを使うときの3パターン

はるさんの実体験・エピソードを投稿に使う場合、「ただの昔話」にしない。必ず以下3つのどれかの目的で使う。

### パターンA：有益投稿型（エピソードを証拠として使う）
- エピソードは「なぜこのアドバイスが正しいか」の証拠として置く
- 構造：「私は〇〇した（エピソード1〜2行）→ そこで気づいたこと・学んだこと → 読者への具体的なアドバイス」
- エピソードより「読者が明日から使える情報」の比重を大きくする
- 例：ラーメン屋面接の話 → 「職場が嫌と仕事が嫌は違う。見極め方はこれ」（アドバイスがメイン）

### パターンB：権威構築型（実績で信頼してもらう）
- エピソードで「なぜこの人の話を聞くべきか」を伝える
- 構造：「〇〇してきた私（実績）→ だからこのアカウントではこういうことを発信していく → フォローしておいて」
- 自己紹介・アカウント紹介・新しいテーマを始めるときに使う
- 例：「エージェント同席面接を経験し・3職場で働き・面接9割合格してきた私が、転職攻略を発信しています」

### パターンC：需要確認型（いいねで需要を測る）
- エピソードをチラ見せして「知りたい人いる？」で反応を見る
- 構造：「〇〇な経験がある（チラ見せ）→ 詳しく知りたい人はいいね・コメントで教えて」
- フォロワーが少ない時期でも使える。需要がわかってから本編を書ける
- 例：「子どもがいながら面接100%合格してきた。秘訣知りたい人いる？いいねで教えて」
- **必ず：** 反応があったら必ず本編を投稿する

## ツリー投稿の設計ルール

**内容が多くてまとめきれないとき、または「続きが気になる」引きを作りたいときに使う。**

- `（以下コメント欄）` より前の文章 = メイン投稿（200文字前後）
- `（以下コメント欄）` より後の文章 = コメント欄への続き（制限なし）
- 区切る場所：「リスト・手順の直前」「一番知りたいことの手前」で切る
- 答えを本投稿で出し切らない。核心はツリーに置く

## 転職サイトアフィリ投稿ルール

- 「〇〇がおすすめ！」の広告感は出さない
- 「実際に使った体験から」という文脈で自然に紹介する
- 複数サービスを比較する形が最も信頼される
- 必ず「#PR」または「（PR）」を明記する

### PR投稿のNG例（絶対やらない）
- ❌「〇〇転職サイトを使えばすぐ転職できます！」（断言しすぎ）
- ❌「登録無料！今すぐ登録を！」（広告臭が強い）
- ❌ 冒頭から転職サイト名を出す（広告感が出る）

### PR投稿のOK例（このパターンで書く）
- 「転職サイト3社使ってみて、正直に違いを教えるね。（解説）→ 私が結局使い続けてるのは〇〇だった」
- 「担当者の当たり外れが大きい話を聞いて、比較登録することにした。（体験）→ 今は〇〇を使ってる」
