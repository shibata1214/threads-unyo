# スレッズ自動運用システム

## このプロジェクトについて
看護師「はる」の**看護師転職特化**Threadsアフィリエイト運用システム。
作業フォルダ：`/Users/mina/看護師転職スレッズ運用`
Claudeが投稿文生成・データ分析・改善提案を担当し、**実際の投稿ははるさんが手動で行う**。

### アカウント情報
- Threads/Instagram ID：haru_nurse_job
- ジャンル：看護師転職（クリニック転職・面接攻略・条件交渉・転職サイト比較）
- マネタイズ：ASP経由の看護師転職サイトアフィリエイト
- 育児ジャンルの過去データは `archive-ikuji/` フォルダに保存済み

### 運用フロー
1. Claude（ライター）が投稿文10本を生成 → `post-queue.md` に追加
2. はるさんが好みの投稿を選んでThreadsに手動投稿（予約投稿機能を使用）
3. Claude（フェッチャー）がエンゲージメントデータを取得
4. Claude（アナリスト）が分析→ルール反映→次ネタ提案

## 重要ファイル
- `01_profile.md` : アカウント概要・コンセプト
- `02_target.md` : ターゲット像・ペルソナ
- `05_writing.md` : 文章ルール・トンマナ・フック型
- `06_references.md` : バズった投稿パターン・競合分析
- `07_ng-rules.md` : 絶対やってはいけないこと
- `next-topics.md` : 次に書くテーマ（アナリストが更新）
- `post-queue.md` : 投稿待機列（ライターが追加）
- `post-history.md` : 投稿記録
- `analysis-latest.md` : 最新分析結果
- `edit-history.md` : ★はるさんの編集傾向まとめ（ライターが最優先で参照）

## 基本ルール
- 投稿文は必ず `05_writing.md` のルールに従う
- `07_ng-rules.md` の内容は絶対に違反しない
- アフィリエイトリンクには必ず「PR」を明記
- ライターは毎日10本生成する（はるさんが選んで投稿する）
- 実際の投稿は1日7本まで（8本以上は凍結リスク）
- **Claudeは投稿を自動送信しない。投稿はすべてはるさんが手動で行う。**

## 【必須】分析→ルール反映ルール

**分析するたびに必ず以下を実行すること。やらないと分析する意味がない。**

### フェッチャーがメトリクスを取得するたびに
1. `06_references.md` の「はるさん自身の投稿データから学んだこと」セクションを更新する
   - 投稿パフォーマンス比較表に新しい数字を追記
   - 勝ちパターン・負けパターンを更新
2. 新しいコメントがあれば `next-topics.md` にネタを追加する

### アナリストが分析するたびに
1. 発見した勝ちパターンを `06_references.md` に追記する
2. 文章・構成の改善ポイントは `05_writing.md` に追記する
3. 「次にやるべきこと」を `next-topics.md` に反映する
4. **反応が良かった投稿（いいね・保存・コメントが多い）は `next-topics.md` に「リメイク候補」として追加する**
   - 同じテーマを切り口・角度・型を変えて再投稿するため
   - 例：型を変える（警告型→体験談型）、ターゲットを変える（0歳ママ→3歳ママ）、数字や事例を別のものにする
5. **反応が良かったPR投稿・PR商品は `next-topics.md` に「7回訴求リスト」として追加する**
   - 切り口プランを7パターン書き出し、残りの本数（7本中何本済みか）を明記する
6. **「（以下コメント欄）」を使った投稿のツリーインプレッション（コメント欄閲覧数）を確認する**
   - 本投稿の20%未満なら切り位置を見直し、`next-topics.md` に「切り位置変更リメイク」として追加する

### ライターが投稿文を作るときに必ず参照するファイル
1. `edit-history.md` ：★★最優先★★ はるさんの編集傾向まとめ（文体・フック・リズムの好み）
2. `05_writing.md` ：文章ルール・フック型・ツリー設計
3. `06_references.md` の **「はるさん自身の投稿データから学んだこと」セクション**（★最優先）
4. `06_references.md` の競合分析・トレンド
5. `next-topics.md` ：テーマ候補

**「はるさん自身のデータ」は競合分析より優先度が高い。自分の投稿で何が当たったかが最重要。**
**「はるさんの編集傾向（edit-history.md）」はその中でも最上位。はるさんが実際に手直しした内容がクオリティの基準。**

### はるさんが投稿を編集して渡してきたとき
`/learn-edit` を実行して、編集前後の差分から傾向を学習し `edit-history.md` と `05_writing.md` を更新する。
