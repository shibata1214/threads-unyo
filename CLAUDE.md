# スレッズ自動運用システム

## このプロジェクトについて
看護師ママ「はる」のThreadsアフィリエイト自動運用システム。
6つのエージェントが連携して投稿生成〜投稿〜分析〜改善をループする。

## 重要ファイル
- `01_profile.md` : アカウント概要・コンセプト
- `02_target.md` : ターゲット像・ペルソナ
- `05_writing.md` : 文章ルール・トンマナ・フック型
- `06_references.md` : バズった投稿パターン・競合分析
- `07_ng-rules.md` : 絶対やってはいけないこと
- `next-topics.md` : 次に書くテーマ（アナリストが更新）
- `post-queue.md` : 投稿待機列（ライターが追加）
- `post-history.md` : 投稿記録
- `analysis-latest.md` : 最新分析結果

## 基本ルール
- 投稿文は必ず `05_writing.md` のルールに従う
- `07_ng-rules.md` の内容は絶対に違反しない
- アフィリエイトリンクには必ず「PR」を明記
- 1日3件以上の投稿は禁止（凍結リスク）

## 【必須】分析→ルール反映ルール

**分析するたびに必ず以下を実行すること。やらないと分析する意味がない。**

### フェッチャーがメトリクスを取得するたびに
1. `06_references.md` の「はるさん自身の投稿データから学んだこと」セクションを更新する
   - 投稿パフォーマンス比較表に新しい数字を追記
   - 勝ちパターン・負けパターンを更新
2. 新しいコメントがあれば `next-topics.md` にネタを追加する

### アナリストが分析するたびに
1. 発見した勝ちパターンを `06_references.md` に追記する
2. 文章・構成の改善ポイントは `05_writing.md` に追記する
3. 「次にやるべきこと」を `next-topics.md` に反映する

### ライターが投稿文を作るときに必ず参照するファイル
1. `05_writing.md` ：文章ルール・フック型・ツリー設計
2. `06_references.md` の **「はるさん自身の投稿データから学んだこと」セクション**（★最優先）
3. `06_references.md` の競合分析・トレンド
4. `next-topics.md` ：テーマ候補

**「はるさん自身のデータ」は競合分析より優先度が高い。自分の投稿で何が当たったかが最重要。**
